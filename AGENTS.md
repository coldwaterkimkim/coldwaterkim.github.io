# Codex Workspace Rules

- 모든 답변은 한국어로 한다.
- 설명은 간결하고 핵심 위주로 한다.
- 작업을 시작할 때 현재 워크스페이스 구조와 관련 파일을 먼저 확인한다.
- 공개 사이트의 UI, 카피, 페이지 구조를 만들거나 바꿀 때는 먼저 `index.html`, `css/styles.css`, `design.md`를 읽고 현재 홈페이지의 90s 개인 홈페이지 감성을 유지한다.
- 공개 사이트의 색상, 폰트, 간격, 움직임 값은 `css/styles.css`의 `:root` 토큰과 `design.md`의 디자인 토큰 표를 기준으로 한다.
- 의도적으로 디자인 방향을 바꾸는 경우에는 `design.md`와 `STATUS.md`도 함께 갱신한다.
